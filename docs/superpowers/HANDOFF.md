## Инварианты — НЕ СЛОМАТЬ

Каждый из них когда-то был реальным багом.

1. **`WORKLET_URL` (`SoundTouchEngine.ts`) — молчаливый убийца.** Под подпутём неверный путь → `SoundTouchNode.register` реджектится → **звука нет вообще, при идеально отрисованном UI**. Ни `tsc` (конфиг вне гейта), ни тесты (**vitest принудительно ставит `base: '/'`**) этого не видят. Защита: `vite.config` падает на кривом `base`, CI грепает боевой бандл. Не ослаблять.
2. **`WaveformCanvas` — единственный владелец `store.tick()`.** MiniMap читает `position`, свой тик не зовёт (два тикера дважды опрашивали бы часы движка).
3. **Оба canvas читают стор императивно** (`store.getState()` в rAF), НЕ через React-селектор: `position` меняется 60/с.
4. **Жесты — только `e.targetTouches`**, никогда `e.touches` (тот считает пальцы по всему экрану → палец на мини-карте ложно включал pinch-зум на волне). Оба canvas слушают `touchcancel`.
5. **`source.onended` — guard по идентичности ноды**, не boolean-флагом (`stop()` шлёт `ended` асинхронно).
6. **Темп = `playbackRate` на source И на node**, оба сразу. Позиция считается формулой, не репортится из worklet.
7. **zustand v5:** мульти-полевой селектор → `useShallow`.
8. **Логику — ВОН из canvas.** Жесты мини-карты давали HIGH-баг в каждой версии, пока жили внутри компонента. Теперь чистый редьюсер + тонкий адаптер. Со следующим жестом — так же.

## Осталось (по приоритету)

### 0. Редизайн библиотеки — следующий по очереди
Плеер переработан, библиотека осталась MVP: `<h1>` инлайн-стилем, строки с `🗑` и нативным `confirm()`, тап открывает трек только по имени. Дизайн-система (`theme.ts` + токены в CSS) уже готова — ложиться на неё. Сюда же едет **UI-переключатель темы**: архитектура (`applyTheme`/`loadThemeName`/`localStorage`) есть и не используется, кнопки нет нигде. Подробности, инварианты и открытые вопросы: **`HANDOFF-library-redesign.md`**.

### 1. Рассинхрон звука и волны — известное ограничение, не случайность
SoundTouch буферизует внутри (FIFO), а наша позиция — чистая формула, описывающая **вход** в worklet, а не **выход в динамик**. Итог: **playhead опережает звук**. Чинится вычитанием калиброванной задержки: `ctx.baseLatency + ctx.outputLatency` (запрашиваются) + внутренний буфер SoundTouch (**надо мерить**). Не угадывается — нужна задача с замерами на устройстве.

### 2. Перезагрузка при возврате на iOS
iOS выгружает фоновую вкладку и выполняет её заново. **Запретить нельзя** — решение ОС; мы жирная мишень, потому что держим декодированный `AudioBuffer` целого трека (5 мин стерео 44.1кГц ≈ **~100 МБ**).
Лечение симптома (в нашей власти): сохранять `currentTrackId` и восстанавливать на `init()` — сейчас после перезагрузки выкидывает в библиотеку. Тогда возврат станет почти незаметным: трек сам откроется на той же позиции.
Полное лечение (не держать весь трек в памяти) — переделка движка, отдельная большая тема.

### 3. Робастность из спеки (раздел «Обработка ошибок»), не закрыто
- Переполнение квоты IndexedDB → catch + «удали старые треки» (`db.ts` без catch).
- Feature-detect AudioWorklet на старом браузере → сообщение (сейчас белый экран).
- Долгий `computePeaks` на больших файлах → чанки + прогресс (сейчас синхронный, морозит UI).
- iOS suspended `AudioContext` → `ctx.resume()` есть, UX «нажми play» нет.

### 4. Rubberband — ЖДЁТ решения
Вторая реализация `AudioEngine` за тем же интерфейсом. Лечит «качество растяжки SoundTouch недостаточно». **Брать только если device-тест подтвердит, что артефакты слышны** — тяжёлый WASM + риск по батарее ради неподтверждённой проблемы это плохая ставка.

## Открытый долг (чекбоксы в `docs/tasks/*.md`, читаются `/debts-report`)

- `openTrack` при `rec == null` молча возвращает без ошибки юзеру.
- `addMarker` не дедуплицирует — повторные тапы дают накладывающиеся маркеры.
- persist→restore маркеров без автотеста (jsdom без AudioContext).
- A-B луп: скраб за пределы `[A,B]` на отпускании защёлкивается обратно в регион.
- MiniMap: размонтаж посреди drag не резюмит воспроизведение (практически недостижимо).
- Нет maskable-иконки → Android залеттербоксит в белый круг.
- wakeLock: гонка async request vs sync release; нет re-acquire на `visibilitychange`.
- Луп при B<A: движок корректно неактивен, но UI рисует регион.
- `db.ts`: singleton `dbp` не сбрасывается при reject `openDB`; `deleteTrack` не атомарен.

## Ключевые уроки этой сессии

- **Device-тест ловит то, что не поймает ничто другое.** 4 бага (`crypto.randomUUID`, iOS-пикер, абсолютный скраб, long-press меню) не видны ни в одном тесте и ни в одном ревью кода — только на живом телефоне.
- **Plan-review окупается на жестах и деплое.** Поймал: дедлок стрипа, протухший `wasPlaying`, отсутствие slop-порога, ложный pinch между canvas'ами, неребейзящийся `href` иконки, UI под часами.
- **Зелёные тесты ≠ работает.** Целые классы отказов (звук, base, жесты, PWA) структурно вне их досягаемости — это записано в CLAUDE.md, не полагаться на CI слепо.

## Процесс

Каждая задача идёт через `run-task-pipeline`: brainstorm → триаж пресета → (plan → plan-review) → impl (TDD) → review (verify + code-review) → done. Task-файлы в `docs/tasks/`, планы в `docs/superpowers/plans/`, спека в `docs/superpowers/specs/`.
