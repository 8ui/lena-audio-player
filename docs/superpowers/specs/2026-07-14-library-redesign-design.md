# Спека — редизайн экрана библиотеки

**Дата:** 2026-07-14
**Хендофф:** `docs/superpowers/HANDOFF-library-redesign.md`
**Предшественник:** редизайн плеера (`2026-07-13-player-ui-redesign-design.md`, смержен как `e9b2820`)

## Задача

`src/screens/Library.tsx` — стартовый экран (список треков + импорт) — остался MVP-шным
и после редизайна плеера визуально выбивается. Переработать его на **существующей**
дизайн-системе (`theme.ts` + токены в `styles.css`), не заводя вторую.

Сценарий тот же, что у плеера: **телефон на пюпитре, читать с 50–70 см**, тыкают редко
и вслепую. Крупные цели, читаемые подписи.

Функциональность (импорт, открытие, удаление) сохраняется. Плюс — три вещи, которые
экран сегодня не делает и должен: показывать **где пользователь остановился**, держать
**заданный порядок** треков, и дать **переключатель темы** (архитектура для него готова
и простаивает).

## Решения

Приняты в брейншторме, не переигрывать без разговора:

| Развилка | Решение |
|---|---|
| Строка или карточка | **Карточка** с мини-волной, прогрессом и бейджами настроек |
| Мини-волна | **SVG-бары из чистой функции** (не канвас) |
| Удаление | Кнопка `⋯` на карточке → **свой боттом-шит**, не `confirm()` |
| Переименование | **Не в этой задаче.** Шит к нему готов — вторая кнопка |
| Импорт | **FAB** внизу справа (плюс CTA на пустом экране) |
| Переключатель темы | **Иконка-свотч в шапке.** Отдельного экрана настроек нет — YAGNI при одной настройке |
| Порядок | **Новые сверху** (`createdAt` DESC). Выбора сортировки и поиска в UI нет — YAGNI при 5–20 треках |
| Палитра по умолчанию | **Warm Graphite**, без изменений |
| Плеер | **Не трогаем** |

### Почему SVG, а не канвас

Канвас на карточку означал бы N императивных рендереров — по CLAUDE.md ровно тот класс
кода, который юнит-тестами не покрывается и проверяется только на устройстве. SVG-бары
считаются **чистой функцией** (тестируется), красятся **CSS-переменными** (переключение
темы перекрашивает карточки само, без строчки кода перерисовки), и `~48 rect × 20 карточек`
для DOM — пыль.

Вариант «отрендерить PNG при импорте и хранить в базе» отпадает: кэш пришлось бы
инвалидировать при смене темы, а тема как раз добавляется здесь.

## Раскладка

```
╭─────────────────────────────────╮
│  Разбор                    ◐    │  .library-header (safe-area-top)
├─────────────────────────────────┤
│  ╭───────────────────────────╮  │
│  │ Осень              3:42 ⋯ │  │  .track-card
│  │ ▁▃▅▇▅▃▁▂▄▆█▆▄▂▁▃▅▇▅▃▁▂▄▆ │  │  волна: пройденное — акцент,
│  │ ▔▔▔▔▔▔▔▔│ 1:24            │  │  остальное — приглушённое
│  │ 90%  −2  A–B              │  │  бейджи — только если не дефолт
│  ╰───────────────────────────╯  │
│  ╭───────────────────────────╮  │
│  │ Прелюдия           5:10 ⋯ │  │
│  │ ▁▂▄▆█▆▄▂▁▃▅▇▅▃▁▂▄▆█▆▄▂▁▃ │  │  не начат → волна вся приглушённая,
│  ╰───────────────────────────╯  │  времени и бейджей нет
│                                 │
│                        ╭─────╮  │
│                        │  ＋ │  │  .import-fab (safe-area-bottom)
│                        ╰─────╯  │
╰─────────────────────────────────╯
```

Скролл — на списке, не на `body`. У списка `padding-bottom` под FAB, иначе последняя
карточка уезжает под кнопку.

Пустой экран — не серый параграф, а центрированный CTA («Выбрать аудио»), открывающий тот
же `<input>`. FAB при этом остаётся на месте: две точки входа в импорт — это нормально,
CTA нужен для обнаружения, FAB — для повторного импорта.

**Тема** — `◐` в шапке: круг, залитый акцентом текущей темы (янтарь / синий). Тап
переключает, круг меняет цвет, и то же изменение немедленно видно по всему экрану.
Подписи «тёплая / студия» не нужны — цвет и есть подпись. Обе палитры тёмные, поэтому
метафора солнце/луна была бы враньём.

## Карточка

Содержимое (сверху вниз):

1. **Имя** (обрезается многоточием) и **длительность** (`fmtTime` — третий форматтер не заводить).
2. **Мини-волна**: `barHeights(peaks, ~48)`. Бары до `lastPosition` — акцентом, после —
   приглушённым; тонкая риска playhead на границе. `lastPosition === 0` → вся волна
   приглушённая, и это само читается как «не начат», без слова «не начат».
3. **Время остановки** (`fmtTime(lastPosition)`) — только когда `lastPosition > 0`.
4. **Бейджи настроек** — только не-дефолтные: темп (`90%`), тон (`−2`), луп (`A–B`).
   Это ответ на «что я с этим треком делал», видимый без открытия.

### Открытая цель тапа

Сегодня обработчик висит на `<span>` с именем, поэтому тап по длительности или по паддингу
строки не делает **ничего**. Чиним честно: карточка — `<div>`, внутри растянутая
`<button className="open">` (`position: absolute; inset: 0`, прозрачный фон — глобальный
`button { background: var(--elevated) }` перебить), контент над ней с `pointer-events: none`,
`⋯` — отдельной кнопкой выше по z-index. Тап **в любую точку** карточки открывает трек.

## Шит действий

`⋯` → бэкдроп + шит снизу: имя трека, **«Удалить»** (красная), **«Отмена»**. Удаление
подтверждается прямо в шите — второго диалога нет, `confirm()` уходит совсем (в
установленной PWA он выглядит как чужеродный системный диалог).

Какой трек открыт — **локальный React-state** `Library` (`sheetTrackId`), не стор. Тот же
принцип, что у `ControlTabs`: какая вкладка открыта — никогда не в сторе.

## Данные

### Чистые модули (сюда уезжает вся логика — инвариант 7)

- **`waveform/computePeaks.ts`** — добавить `barHeights(peaks, bars): number[]`, амплитуда
  0..1 на бар, поверх готового `downsamplePeaks`. Тесты — в существующий
  `computePeaks.test.ts`.
- **`ui/libraryModel.ts`** (новый) + `libraryModel.test.ts`:
  - `sortTracks(tracks)` → копия, `createdAt` DESC;
  - `progressRatio(lastPosition, duration)` → 0..1, устойчиво к `duration === 0`;
  - `tempoBadge(1) → null`, `tempoBadge(0.9) → '90%'`;
  - `pitchBadge(0) → null`, `pitchBadge(-2) → '−2'`;
  - `loopBadge(a, b) → 'A–B' | null`.

  Карточка не считает **ничего** — только рисует то, что ей дали.

### Хранилище (`storage/db.ts`)

- `listStates(): Promise<TrackStateRecord[]>` → `getAll('trackState')`.

Переполнение квоты IndexedDB сегодня глотается молча (долг из `HANDOFF.md`): импорт
большого файла на забитом телефоне просто ничего не делает. Ловим это **в сторе**, вокруг
вызова `db.addTrack`, а не внутри `db.ts` — там отказ пинится тестом
(`vi.spyOn(db, 'addTrack').mockRejectedValue(...)` → проверяем баннер), а на
`fake-indexeddb` квоту не подделать и тест был бы фиктивным. Баннер ошибки уже есть.

### Стор (`store/usePlayerStore.ts`)

Новое поле `trackStates: Record<string, TrackStateRecord>`:

- `init()` — читает `listTracks()` и `listStates()`;
- `importFile()` — обновляет `library` (у нового трека состояния ещё нет → дефолт);
- `removeTrack()` — удаляет ключ из мапы;
- `closeTrack()` — **после** `flushPersist()` кладёт текущее состояние в `trackStates[id]`
  локально. Это то, что даёт свежий прогресс сразу по возврату из плеера, без повторного
  чтения базы. `id` надо захватить до того, как `currentTrackId` обнулится.

`Library` селектит `library` и `trackStates` **раздельными однополевыми селекторами** —
не одним объектным. Иначе гоча 1 (zustand v5, «getSnapshot should be cached»).

### Тема (`ui/theme.ts`)

Умеет читать (`loadThemeName`) и применять (`applyTheme`), но **писать не умеет** —
добавить `setThemeName(name)`: запись в `localStorage` (ключ `razbor.theme`) + `applyTheme`.
`ThemeToggle` держит имя темы в локальном `useState`; стор про тему не знает.

Канвасы плеера читают палитру покадрово (`activePalette()`), поэтому подписка им не нужна —
при возврате в плеер волна уже в новом цвете.

## Слои (гоча 9, здесь перевёрнута)

В плеере ряды дока намеренно **выше** бэкдропа. Здесь — наоборот: бэкдроп шита обязан быть
**выше шапки и FAB**, иначе тап по `＋` при открытом шите откроет файловый пикер вместо
того, чтобы закрыть шит.

```
.library-header, .import-fab   z-index: 30
.sheet-backdrop                z-index: 40
.sheet                         z-index: 50
```

jsdom хит-тест стековых контекстов не делает (RTL кликает прямо по узлу), поэтому порядок
**пинится тестом, читающим `styles.css` через `?raw`** — ровно как в `ControlTabs.test.tsx`.

## Инварианты, которые здесь легко нарушить

1. **`AUDIO_ACCEPT` переезжает в `ImportFab.tsx` дословно.** Не упрощать до `audio/*`: iOS
   мапит `accept` на UTI и **гасит mp3 в пикере** — импорт становится физически невозможен.
   Экспорт и тест на него сохранить.
2. **Тач-цели ≥ 48px** (`--touch`): `⋯`, FAB, кнопки шита, свотч темы.
3. **safe-area**: шапка — `env(safe-area-inset-top)`, FAB и шит — `env(safe-area-inset-bottom)`.
4. **zustand v5**: однополевые селекторы, либо `useShallow`.
5. **Логика — вон из компонентов**: сортировка, прогресс, бейджи, высоты баров — чистые
   функции с тестами.

## Тесты

**Юниты:** `barHeights` (в `computePeaks.test.ts`), `libraryModel.test.ts` (сортировка,
прогресс, бейджи, деление на нулевую длительность).

**RTL (`Library.test.tsx`):**
- список рендерится в порядке «новые сверху»;
- тап **по краю карточки** (не по имени) открывает трек — это регрессия на найденный баг;
- `⋯` открывает шит; «Удалить» зовёт `removeTrack`; «Отмена» — не зовёт;
- пустая библиотека показывает CTA;
- `AUDIO_ACCEPT` не выродился (тест сохранить как есть).

**RTL (`ThemeToggle.test.tsx`):** тап пишет `localStorage` и `data-theme` на `<html>`.

**CSS (`layers.test.ts`):** `z-index` бэкдропа шита строго больше, чем у шапки и FAB.

**Чего тесты структурно не поймают** — только устройство:
- читается ли волна карточки с 50–70 см;
- не съедает ли бэкдроп первый тап;
- не села ли шапка под чёлку, не сел ли FAB на home indicator;
- переключилась ли тема **вместе с волной в плеере**;
- реально ли открывается mp3 в пикере на iPhone.
