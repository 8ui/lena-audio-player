# Переработка интерфейса плеера

**Дата:** 2026-07-13
**Статус:** утверждено, готово к планированию реализации

## Цель

Экран плеера сейчас — MVP: семь равнозначных блоков в вертикальном стеке, одинаковые
серые кнопки, 40 строк CSS, никакой иерархии. Приложение используют, поставив телефон
на пюпитр (инструмент в руках), поэтому читаемость с 50–70 см и крупные цели важнее
плотности.

Переработка охватывает **визуал и раскладку**. Функциональность, жесты, движок, стор и
схема IndexedDB не меняются.

### Приоритет элементов на экране

1. Темп (главная ручка при разборе на слух)
2. Play/pause
3. Волна + минимапа

Второстепенное (тон, A-B луп, маркеры) уходит в табы с поповером.

## Скоуп

**В скоупе:** экран плеера (`src/screens/Player.tsx`, `src/ui/*`), палитра канвасов
(только цвета — не логика), дизайн-токены, тема.

**Вне скоупа:** экран библиотеки (отдельная задача; общие токены не должны его сломать),
UI-переключатель темы (приедет вместе с библиотекой), любые изменения движка, стора,
жестов, схемы БД.

## Раскладка

Снизу вверх экран плеера состоит из:

```
┌─────────────────────────────┐
│ ‹ Библиотека     track.mp3  │  PlayerHeader (тонкий)
├─────────────────────────────┤
│                  ┌─────────┐│
│                  │1:24/3:58││  TimeBadge — оверлей поверх волны,
│      волна       └─────────┘│  справа сверху, pointer-events: none
│                             │  WaveformCanvas: flex: 1
├─────────────────────────────┤
│ минимапа                    │  MiniMap: 48px, без изменений
├─────────────────────────────┤
│  [ − ]  [ 0.75× ]  [ ＋ ]   │  TempoStepper
│           ( ▶ )             │  TransportBar — только play, по центру
│  [♪ Тон 0][A–B ✓][⚑ 3]      │  ControlTabs
└─────────────────────────────┘  PlayerDock: sticky, safe-area-inset-bottom
```

**Убрано:** слайдер темпа, кнопки −5 / +5, текст времени в транспорте.

**TimeBadge** — маленькая плашка (текущее / общее время) поверх волны справа сверху:
скруглённый контейнер, фон чуть светлее фона канваса, `pointer-events: none`, чтобы
жест по волне под ней проходил насквозь.

## Темп

Три кнопки: `[−] [значение] [＋]`. Шаг **0.1**, диапазон существующий (`TEMPO_MIN = 0.25`,
`TEMPO_MAX = 1.5`). Тап по центральной кнопке (она же показывает текущее значение) —
сброс на `1.00×`.

Шаг даёт float-мусор (`0.75 − 0.1 = 0.6499999…`), поэтому в `src/engine/params.ts`
появляется чистая функция:

```ts
export const TEMPO_STEP = 0.1;
export const stepTempo = (t: number, dir: 1 | -1): number =>
  clampTempo(Math.round((t + dir * TEMPO_STEP) * 100) / 100);
```

Юнит-тесты — рядом с `clampTempo` (`params.test.ts`): округление, клампы на обеих
границах, идемпотентность на краю диапазона.

## Табы и поповер

`ControlTabs` — три чипа (Тон / A–B / Маркеры), которые ведут себя как табы:

- Чип всегда показывает **состояние**: `♪ Тон 0`, `A–B ✓` (подсвечен, когда луп задан),
  `⚑ 3` (число маркеров).
- Тап открывает поповер с контролами этого таба. Поповер — **оверлей поверх интерфейса
  (`position: absolute`), раскладку не двигает**, волна не ужимается.
- При открытом поповере чипы остаются видимыми и кликабельными: тап по соседнему
  переключает содержимое, тап по активному — закрывает. Тап вне поповера — закрывает.
- Все три панели одной высоты, чтобы переключение табов не дёргало раскладку.

Содержимое панелей (это текущие контролы, только перерисованные):

| Таб | Панель |
|---|---|
| Тон | `[−] [+2] [＋]` — полутона, существующий `setPitch` |
| A–B | `[A 0:42] [B 1:08] [Сброс]` |
| Маркеры | `[◀] [＋ маркер] [▶] [−]` |

Открытый таб — **локальный `useState` в `ControlTabs`** (`'pitch' | 'loop' | 'markers' | null`).
Это UI-состояние: в стор и в IndexedDB оно не попадает.

## Тема и токены

Цвета сейчас живут в двух несвязанных местах: CSS (`styles.css`) и канвасы (строковые
`fillStyle`/`strokeStyle` в JS). При янтарной теме и синей волне приложение выглядит
сломанным, поэтому нужен один источник.

**`src/ui/theme.ts`** — единственный источник правды:

```ts
export type ThemeName = 'warm' | 'studio';
export interface Palette {
  bg: string; surface: string; elevated: string; border: string;
  text: string; muted: string;
  accent: string;      // play, значение темпа, пики волны
  playhead: string;    // «сейчас»
  loopFill: string;    // заливка региона лупа (rgba)
  marker: string;      // маркеры
  canvasBg: string;
  minimapPeaks: string; // приглушённые пики минимапы
}
export const THEMES: Record<ThemeName, Palette>;
```

Канвасы сегодня рисуют регион лупа только заливкой (`fillRect`), без обводки, — поэтому
отдельного токена под границы A-B нет, `loopFill` покрывает всё.

Две палитры:

**Warm Graphite (по умолчанию)** — тёплый графит + янтарь, pill-кнопки:
`bg #17150f`, `surface #1e1b14`, `elevated #241f18`, `border #2b2720`,
`text #f2ece0`, `muted #a09582`, `accent #ffb43f`, `playhead #ff5a5a`,
`marker #8be0ff`, `loopFill rgba(255,180,63,0.14)`, `canvasBg #100f0b`,
`minimapPeaks #8a6a30`.

Смысловое разделение цветов на канвасе: **янтарь = данные (волна), красный = «сейчас»
(плейхед), бирюза = маркеры**.

**Studio Dark** — нынешние цвета, поднятые в токены:
`bg #12141a`, `surface #181b22`, `elevated #2a2e3a`, `border #23262f`,
`text #eeeeee`, `muted #8b93a7`, `accent #5aa0ff`, `playhead #ff5a5a`,
`marker #ffcf5a`, `loopFill rgba(90,160,255,0.18)`, `canvasBg #0c0e12`,
`minimapPeaks #3f6ea8`.

Механика:

- Активная тема пишется в `<html data-theme="warm|studio">`; `styles.css` объявляет
  CSS-переменные в блоках `:root[data-theme="warm"]` / `:root[data-theme="studio"]`.
- Канвасы (`WaveformCanvas`, `MiniMap`) импортируют палитру из `theme.ts` и берут цвета
  из активной темы. Перерисовка бесплатна: обе канвы и так рисуют каждый кадр.
- Выбор темы хранится в `localStorage` (настройка UI, не данные трека — в IndexedDB ей
  не место). Дефолт — `warm`.
- **UI-переключателя в этой задаче нет.** Проверить Studio на устройстве можно, выставив
  ключ в `localStorage`. Переключатель приедет вместе с экраном библиотеки.

**Дрейф значений** между `theme.ts` и `styles.css` ловится юнит-тестом: тест читает
`styles.css`, парсит `:root[data-theme=…]` блоки и сравнивает с `THEMES`. Дёшево, и
ловит ровно ту ошибку, которую иначе заметит только глаз на устройстве.

## Риски

1. **Янтарный вош лупа поверх янтарной волны** может читаться хуже, чем нынешний синий
   поверх синей. Осознанное решение: сначала как на утверждённом макете, при провале на
   устройстве — сменить `loopFill` на бирюзовый (одна константа в `theme.ts`).
2. **Волна `flex: 1` вместо `50vh`** — высота канваса теперь зависит от дока. Канвас уже
   пересоздаёт backing store при смене размера, но проверить на устройстве надо
   (особенно при появлении поповера — он оверлей, размер канваса менять не должен).
3. **Токены ломают библиотеку.** Библиотека вне скоупа, но использует те же `body`/`button`
   стили. После правок открыть её и убедиться, что она не развалилась.
